PRD - PhoAmThuc (Vinh Khanh Audio Guide)

Tài liệu Product Requirements cho đồ án: hệ thống quản trị nội dung audio guide ẩm thực với backend API, dashboard admin và các luồng analytics vận hành.

ASP.NET Core Minimal API EF Core + SQLite React + Vite Admin POI / Audio / Tour Session + Analytics QR + Geofence + Route
🎯
Section 01

Tổng quan sản phẩm

Sản phẩm giải quyết bài toán quản trị và phân phối nội dung audio theo điểm đến (POI), đồng thời cung cấp dữ liệu đo lường để tối ưu trải nghiệm tham quan ẩm thực.

Bài toán

Thông tin ẩm thực phân tán, khó cập nhật, không đo được mức độ sử dụng nội dung theo điểm tham quan.

Tầm nhìn

Xây dựng nền tảng audio guide có thể quản trị tập trung, phát nội dung linh hoạt qua QR/GPS và có dữ liệu analytics để vận hành.

Đối tượng

  • Admin vận hành nội dung.
  • Người dùng cuối (qua mobile app tích hợp API).
📦
Section 02

Phạm vi sản phẩm (In Scope / Out of Scope)

In Scope (MVP hiện tại)

  • CRUD POI (priority, imageUrl, mapLink).
  • CRUD audio theo POI + upload multipart.
  • Tour và reorder điểm dừng.
  • Translation đa ngôn ngữ + fallback.
  • Resolve user, activate subscription, segment access.
  • Session tracking + analytics top/usage/heatmap.
  • QR playback và anonymous route tracking.
  • Dashboard bản đồ cho admin.

Out of Scope (đợt đồ án này)

  • Thanh toán thật (payment gateway).
  • AuthN/AuthZ production-grade hoàn chỉnh.
  • ShopManager module.
  • SQL Server large-data mode.
  • Kiến trúc multi-tenant nâng cao.

Tech Stack chính

  • Backend: .NET 10, ASP.NET Core, EF Core 10, SQLite.
  • Frontend: React 19, Vite 5, React Router.
  • Map & QR: Leaflet, react-leaflet, qrcode.react.
  • Test: xUnit + EF InMemory/SQLite.
Ghi chú phạm vi: Repo hiện tại tập trung backend và web admin; mobile app MAUI được xem là client tiêu thụ API ở ngoài phạm vi codebase này.
📋
Section 03

Yêu cầu sản phẩm

Functional Requirements (FR)

Mã Nhóm Yêu cầu
FR-01..03 User & Access Resolve user theo externalRef, activate subscription, kiểm tra access theo segment.
FR-04..07 POI CRUD POI, code unique, có metadata mở rộng (district, priority, imageUrl, mapLink).
FR-08..10 Audio CRUD audio theo POI, upload multipart, trả filePath/relativePath cho frontend.
FR-11..13 Tour CRUD tour, thêm stop, reorder theo sequence.
FR-14..17 Session & Analytics Ghi nhận start/end session, usage theo ngày, top POI, heatmap theo thời gian.
FR-18..20 QR & Route Xử lý payload QR, ghi route theo anonymousRef, trả route để render polyline.
FR-21..23 Translation & Sync CRUD translation, fallback language, snapshot sync nội dung.

Non-functional Requirements (NFR)

Hiệu năng

  • Endpoint đọc phổ biến < 500ms (local demo).
  • Dashboard tải tổng quan < 2 giây với dữ liệu demo.

Độ tin cậy

  • Có health endpoint để smoke test.
  • Seed idempotent, restart không nhân bản dữ liệu.

Bảo trì & bảo mật

  • Logic nghiệp vụ tập trung ở service layer.
  • Validate input và giới hạn loại file upload hợp lệ.
  • Có lộ trình bổ sung auth production.

Section 04

Luồng nghiệp vụ chính (End-to-End)

Luồng người dùng nghe nội dung

/users/resolve → /subscriptions/access → /pois + /audios → /sessions/start-end

Dữ liệu session sau khi kết thúc sẽ được tổng hợp lên analytics để phục vụ dashboard admin.

Luồng QR playback

  1. Admin tạo QR payload theo chuẩn QR:POI_CODE.
  2. App quét QR và gọi POST /api/v1/qr/start.
  3. Backend parse payload, xác định POI, chuẩn bị dữ liệu phát.

Luồng admin vận hành nội dung

  1. Tạo/cập nhật POI.
  2. Upload audio và gán ngôn ngữ.
  3. Tạo tour và sắp xếp stop.
  4. Cập nhật translation.
  5. Quan sát dashboard để tối ưu.

Use Cases trọng tâm

Use Case Tác nhân Kết quả mong đợi
UC-01 Quản trị POI Admin POI hiển thị đúng trên danh sách và dashboard.
UC-02 Upload/Gán audio Admin Audio truy cập được qua media URL và liên kết đúng POI.
UC-03 Tạo tour/reorder Admin Thứ tự tour stop phản ánh chính xác trên API.
UC-04 Session start/end Mobile app Có dữ liệu đo lường dùng cho analytics.
UC-05 QR start Mobile app Điều hướng đúng POI theo payload.
UC-06 Dashboard analytics Admin Hiển thị top/usage/heatmap/route phục vụ vận hành.

Section 05

Kế hoạch triển khai và roadmap

Milestone 1 - Core API

Hoàn thiện users, subscriptions, POI, audio, tours và cơ chế init/seed database.

Milestone 2 - Tracking & Dashboard

Hoàn thiện sessions, analytics, anonymous routes và dashboard quản trị trực quan.

Milestone 3 - Vận hành nội dung

Bổ sung translation, content sync, QR manager, rà soát test/build và tài liệu demo bảo vệ.

Mở rộng sau đồ án Auth đầy đủ (JWT + role/permission), object storage cho media.
Nâng cấp dữ liệu Chuyển chiến lược migration versioned thay cho patch runtime.
Quan sát hệ thống Bổ sung tracing, metrics, centralized logging.
Business nâng cao Tích hợp payment gateway thật cho subscription.
✅
Section 06

Tiêu chí nghiệm thu

Nghiệm thu chức năng

  • CRUD POI/audio hoạt động đúng.
  • Upload audio trả URL hợp lệ và truy cập được.
  • Tour reorder phản ánh đúng sequence.
  • Session start/end ghi nhận dữ liệu chính xác.
  • Dashboard hiển thị top/usage/heatmap/route.

Nghiệm thu kỹ thuật

  • dotnet build PhoAmThuc.sln thành công.
  • dotnet test PhoAmThuc.sln pass.
  • npm run build ở admin thành công.
  • scripts/run-dev.ps1 mở backend + frontend ổn định.

KPI demo đề xuất

  • 100% nhóm endpoint cốt lõi có thể demo.
  • Health endpoint duy trì ổn định suốt buổi bảo vệ.
  • Thời gian start local hệ thống <= 3 phút.
  • Tỷ lệ test backend pass >= 90% tập test hiện có.
