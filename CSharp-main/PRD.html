PRD — PhoAmThuc (Vinh Khanh Audio Guide)

Product Requirements Document cho đồ án: hệ thống audio guide ẩm thực tự động với Backend API (ASP.NET Core), Admin Web (React) và Mobile App (.NET MAUI). Bao gồm quản lý POI/audio/tour/QR, session tracking, analytics, subscription Basic/Premium, và đa ngôn ngữ.

ASP.NET Core Minimal API EF Core + SQLite React + Vite Admin .NET MAUI Mobile App Leaflet Maps Session + Analytics QR Playback + Geofence Subscription Basic/Premium
🎯
Section 01

Tổng quan sản phẩm

Hệ thống audio guide ẩm thực tự động dành cho khu vực Phố Cổ Vĩnh Khách (Vĩnh Hội, Khánh Hội, Xóm Chiếu). Hệ thống gồm Backend API quản lý nội dung và theo dõi analytics, Admin Web (React) để quản trị nội dung, và Mobile App (MAUI .NET) để người dùng trải nghiệm nghe audio guide qua bản đồ, danh sách POI hoặc quét mã QR.

Bài toán

Thông tin ẩm thực phân tán, khó cập nhật, không đo được mức độ sử dụng nội dung theo điểm tham quan. Người dùng không có cách thuận tiện để nghe audio guide tại mỗi điểm.

Giải pháp

Nền tảng tập trung: Admin quản lý POI/audio/tour/QR, Backend track session và analytics, Mobile App phát audio qua bản đồ hoặc QR với subscription Basic/Premium.

Kiến trúc 3 lớp

  • Backend API: ASP.NET Core Minimal API + EF Core + SQLite
  • Admin Web: React 19 + Vite + TailwindCSS
  • Mobile App: .NET MAUI (Android/iOS/Windows)
Seed data thực tế: 12 POIs (3 quận), 3 tours, audio tiếng Việt + Anh cho mỗi POI, 4 tài khoản mặc định (admin/owner1/owner2/owner3), 4 feature segments.

Phạm vi sản phẩm (In Scope / Out of Scope)

In Scope (MVP hiện tại)

  • Backend API đầy đủ: Users, Subscriptions, POI, Audio, Tours, Sessions, Analytics, QR, Geofence, Route, Translations
  • Admin Web 9 trang: Dashboard, Analytics, POI, Audio, Translation, Tour, QR, Usage History, Subscription
  • Mobile App (MAUI): MainPage, Map, POI List, POI Detail, Audio Player, Tour, QR Scan, Login/Register, Settings
  • CRUD POI/audio/tour/translation/QR + upload audio multipart
  • Session tracking + analytics (top POI, usage, heatmap, live sessions)
  • QR playback + anonymous route tracking + geofence auto-play
  • Subscription Basic/Premium + segment-based access control
  • Đa ngôn ngữ: tiếng Việt + tiếng Anh cho audio & translation
  • Đăng ký/đăng nhập user + feature gating
  • Dashboard bản đồ Leaflet cho admin (marker + heatmap + route polyline)
  • Shop Manager: quản POIs/audio/QR của riêng mình

Out of Scope (đợt đồ án này)

  • JWT authentication đầy đủ (production-grade)
  • Object storage thật cho media files (dùng local filesystem)
  • Versioned migration strategy (EF Core auto-migrate)
  • Observability: tracing, metrics, centralized logging
  • Payment gateway thật cho subscription
  • Multi-language audio ngoài tiếng Việt + Anh
  • Push notifications

Tech Stack chính

  • Backend: .NET 10, ASP.NET Core Minimal API, EF Core 10, SQLite
  • Admin Web: React 19, Vite 5, TailwindCSS, React Router 6, Leaflet
  • Mobile: .NET MAUI, CommunityToolkit.Mvvm, Plugin.Maui.Audio, ZXing.Net.MAUI
  • Test: xUnit + EF Core InMemory
Phạm vi đầy đủ: Repository bao gồm cả Backend API, Admin Web, và Mobile App (MAUI). Tất cả 3 thành phần đều hoạt động, có seed data, sẵn sàng demo.

Section 03

Yêu cầu sản phẩm

Functional Requirements (FR)

Mã Nhóm Yêu cầu
FR-01..03 User & Access Đăng ký tài khoản, resolve user để đăng nhập, phân quyền theo vai trò (Admin/ShopManager/EndUser).
FR-04..06 Subscription Activate subscription (Basic/Premium), kiểm tra access theo segment, admin CRUD subscription.
FR-07..10 POI CRUD POI, code unique, metadata (district, priority, imageUrl, mapLink, triggerRadius). ShopManager chỉ quản POIs của mình.
FR-11..14 Audio Upload multipart mp3, gán audio theo POI và ngôn ngữ (vi/en), xóa audio.
FR-15..18 Tour CRUD tour, thêm stop, reorder stop theo sequence, hiển thị điểm dừng.
FR-19..21 QR & Playback Quét QR → parse payload → kiểm tra subscription → trả audio URL + tạo session. Tạo + download + in QR PNG, export CSV payload.
FR-22..24 Session & Visit Ghi nhận start/end listening session, visit session (với GPS, trigger source), cập nhật audio stats cho visit.
FR-25..28 Analytics Top POIs theo lượt nghe, usage stats theo ngày, heatmap GPS, live active sessions, biểu đồ chi tiết.
FR-29..31 Geofence & Route Đánh giá vùng phủ POI theo GPS, ghi điểm GPS anonymous, lấy route để render polyline trên bản đồ.
FR-32..34 Translation CRUD translation theo key + ngôn ngữ, fallback language, sync snapshot nội dung.

Non-functional Requirements (NFR)

Hiệu năng

  • Endpoint đọc phổ biến < 500ms (local demo)
  • Dashboard tải tổng quan < 2 giây với seed data
  • Audio streaming từ local filesystem

Độ tin cậy

  • Health endpoint để smoke test
  • Seed idempotent, restart không nhân bản dữ liệu
  • Auto-migration khi schema thay đổi

Bảo trì & bảo mật

  • Logic nghiệp vụ tập trung ở service layer (Clean Architecture)
  • Validate input + giới hạn loại file upload (.mp3, .wav, .ogg)
  • Password hash SHA-256 với salt
  • X-User-Id header để xác thực API requests

Section 04

Luồng nghiệp vụ chính (End-to-End)

Luồng người dùng nghe audio

/users/resolve → /qr/start → /media/audio/* → /sessions/end

Người dùng bấm nghe (từ POI detail, list, hoặc map). App resolve user → gọi qr/start → tải audio về cache → phát. Khi kết thúc gọi sessions/end để ghi duration. Dữ liệu tổng hợp lên analytics dashboard.

Luồng QR playback

  1. Admin tạo QR payload theo chuẩn vk://poi/{POI_CODE} (QR Manager)
  2. App quét QR → gọi POST /api/v1/qr/start
  3. Backend parse payload, kiểm tra subscription, tạo ListeningSession, trả audio URL
  4. App phát audio và ghi nhận duration

Luồng admin quản lý nội dung

  1. Admin/ShopManager đăng nhập (admin/1, owner1/1...)
  2. Tạo/cập nhật POI (với tọa độ, hình ảnh, map link)
  3. Upload audio mp3 → gán cho POI theo ngôn ngữ (vi/en)
  4. Cập nhật translation (bản dịch name, description)
  5. Tạo tour và sắp xếp thứ tự stops
  6. Tạo QR codes → tải PNG / in tem QR
  7. Quan sát Dashboard và Analytics để tối ưu nội dung

Luồng Shop Manager

  1. ShopManager đăng nhập với tài khoản owner1/owner2/owner3
  2. Chỉ thấy và quản lý POIs có ManagerUserId = mình (4 POIs mỗi người)
  3. Upload audio, tạo QR cho POIs của mình
  4. Xem analytics chi tiết cho POIs của mình

Use Cases trọng tâm

Use Case Tác nhân Kết quả mong đợi
UC-01 Quản trị POI Admin, ShopManager POI hiển thị đúng trên danh sách, map và dashboard. ShopManager chỉ thấy POIs của mình.
UC-02 Upload/Gán audio Admin, ShopManager Audio truy cập được qua media URL và liên kết đúng POI theo ngôn ngữ.
UC-03 Tạo tour/reorder stops Admin Thứ tự tour stop phản ánh chính xác trên API và trong mobile app.
UC-04 Nghe audio (có login) EndUser Audio phát đúng POI, session ghi nhận dưới userId của user đã đăng nhập.
UC-05 Nghe audio (không login) EndUser App tự động resolve demo user, audio phát bình thường, session ghi nhận dưới demo.
UC-06 QR scan EndUser Quét QR → parse payload → phát audio đúng POI → ghi session.
UC-07 Dashboard analytics Admin, ShopManager Hiển thị top/usage/heatmap/route/live sessions phục vụ vận hành.
UC-08 Subscription Basic/Premium Admin Tạo subscription cho user, user có quyền truy cập tính năng tương ứng.

Section 05

Kế hoạch triển khai và roadmap

Milestone 1 - Nền tảng Backend

Hoàn thiện Users, Subscriptions, POI, Audio, Tours và cơ chế init/seed database. Cơ sở dữ liệu với 12 POIs, 3 Tours, audio mẫu.

Milestone 2 - Admin Web

Hoàn thiện 9 trang admin: Dashboard với bản đồ Leaflet, Analytics, POI Manager, Audio Manager, Translation Manager, Tour Manager, QR Manager, Usage History, Subscription Manager.

Milestone 3 - Mobile App (MAUI)

Hoàn thiện app di động: bản đồ, danh sách POI, trình phát audio, tour, quét QR, đăng nhập/đăng ký. Tích hợp GPS, auto-play geofence.

Milestone 4 - Tracking & Analytics

Hoàn thiện session tracking, visit tracking, geofence, route tracking, analytics dashboard với live sessions, heatmap, top POIs.

Sau đồ án - Auth đầy đủ JWT + refresh token, role-based permission, secure password storage
Sau đồ án - Object Storage S3/Azure Blob cho media files thay vì local filesystem
Sau đồ án - Migration Strategy Chuyển từ auto-migrate sang versioned migration
Sau đồ án - Payment Tích hợp payment gateway thật cho subscription

Rủi ro và phương án giảm thiểu

Rủi ro Tác động Giảm thiểu
Audio files thiếu trong seed POI không phát được audio Seed tạo audio placeholder; admin có thể upload file mới
Session ghi sai userId Analytics không chính xác AppConfig.ResolveDefaultUserIdAsync() đã fix — ưu tiên SecureStorage → X-User-Id header → demo fallback
Lỗi upload media do cấu hình path Không phát được audio Chuẩn hóa thư mục media + trả relativePath rõ ràng
Chưa có JWT authentication đầy đủ Rủi ro khi triển khai thật Đã có X-User-Id header; JWT nằm trong roadmap sau đồ án
SQLite hạn chế tải lớn Khó scale production Thiết kế service/persistence tách lớp, sẵn sàng đổi provider
